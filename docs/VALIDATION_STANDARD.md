# VALIDATION_STANDARD

## Required Checks

DS-012 Visual Regression Policy: every sprint that changes presentation should capture before-and-after phone screenshots for Gameplan, Personnel, Top Plays, Matchups, and Recruiting.

Validation must confirm no raw object coercion, no `undefined`, no literal `null`, no horizontal overflow safeguards removed, static GitHub Pages compatibility, and preservation of Pack A/B/C architecture.

Presentation-only sprints must not modify football scoring logic, player logic, play logic, recommendation logic, or JSON ownership.
